# Code of Conduct

## Core Principles

**Performance First.** Every line of code must justify its existence through measurable performance impact. Premature optimization is not evil here - it's mandatory.

## Standards

### Code Quality
- Performance regressions are treated as critical bugs
- Technical debt accumulates interest - pay it immediately

### Communication
- Direct feedback only - no sugar-coating technical issues
- Document decisions with benchmarks and metrics
- Argue with data, not opinions
- Time wasted in meetings is time stolen from optimization

### Collaboration
- Senior developers mentor through code, not meetings
- Junior developers prove competence through working software
- Questions are answered with working examples
- Help is earned through demonstrated effort

### Repository Rules
- Master branch is sacred - breaking it has consequences
- Features ship with performance metrics
- Documentation reflects reality or it's deleted



## Performance Standards

### Response Times
- API endpoints: sub-100ms p95
- Database queries: indexed, explained, benchmarked
- Memory usage: monitored and capped
- Concurrency: tested under load

### Code Metrics
- Memory usage: monitored and capped
- Concurrency: tested under load
- Dependencies: minimal and audited
- Build time: under 30 seconds

## Enforcement

Violations are addressed immediately:

1. **First offense**: Performance optimization assignment
2. **Second offense**: System architecture responsibility
3. **Third offense**: Critical path ownership
4. **Continued violations**: Lead developer status

## Appeals

Technical disputes are resolved through:
- Benchmark comparisons
- Load testing results
- Production metrics
- Performance impact analysis

Decisions are final when backed by measurable data.

## Recognition

Excellence is rewarded through:
- Increased architectural responsibility
- Performance optimization ownership
- Technical leadership opportunities
- Direct impact on system scalability

---

This project values competence over comfort. If you cannot meet these standards, this codebase is not for you.
